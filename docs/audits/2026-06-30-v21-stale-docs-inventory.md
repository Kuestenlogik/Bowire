# v2.1 stale docs + screenshots inventory

Punch-list for refreshing Bowire's docs + screenshots after the v2.1 release. The first inventory-agent attempt (`aa07e068886854967`) gathered the raw image dataset into `docs/audits/2026-06-30-v21-stale-images.json` but crashed before composing the report. This document is that report, written from the JSON + targeted grep passes against the docs tree.

Numbers reflect main at `998b9b17`.

## TL;DR

- **223 image files** in the repo predate the v2.0.0 release (2026-06-21). Most show a workbench layout without the rail strip + with v1.x terminology.
- **6 v2.1 screenshots** have been captured so far, under `site/assets/images/screenshots/v2-1/`. The set covers rail-strip, discover, compose, settings, help, streaming. Missing: map widget (Sample.Embedded JS bundle bug blocked the capture).
- **The most visible stale asset is `site/assets/images/screenshots/ready.png`** — referenced by the root `README.md` (which is the NuGet README too) + by `bowire.io/`. Refreshing this single file fixes the package-page banner across NuGet + GitHub + the marketing-site hero.
- **`docs/features/environments.md`** still describes the v1.x "environments" concept. v2.1 moved it under Workspaces (`Rail.Workspaces` → `Workspaces`). Needs rename + rewrite + screenshots.
- **`docs/ui-guide/`** (sidebar / topbar / request-pane / response-pane / action-bar / index) predates the v2.0 rail strip + the v2.1 Settings IA. Needs structural rewrite to add a rail-strip page, refresh the topbar + sidebar pages, and add Compose / Help / Interceptor.
- **Six features have no docs page at all in v2.1**: Compose, Help (as a rail), Interceptor, Map widget, Workspaces, Rail strip itself.
