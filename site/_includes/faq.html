Why not just use ***?

Pick the tool you’re coming from.

Swagger UI is great for REST documentation. Bowire keeps that surface and adds everything Swagger doesn’t do: gRPC, GraphQL, SignalR, WebSocket, MCP, MQTT, Socket.IO, OData — all with auto-discovery. No swagger.json to maintain, no protocol-specific tool to switch to when you leave REST.

Already on Swashbuckle? Keep it. Bowire reads your existing OpenAPI document and layers every other protocol on top.

Microsoft’s built-in OpenAPI support in .NET 9+ generates the spec — and only the spec. There’s no UI to actually call your endpoints, no form generator, no streaming view. You still need to pair it with something like Scalar or Swagger UI to get an interactive client, and neither of those cover anything beyond REST.

Bowire is that interactive layer — it reads the OpenAPI document Microsoft generates and gives you the full request/response workbench on top, plus gRPC, SignalR, GraphQL, WebSocket, MQTT, MCP, etc. for the rest of the surface.

Postman is a workbench for collections you curate by hand. Bowire discovers your API live — gRPC via Server Reflection, REST via OpenAPI, GraphQL via introspection, SignalR hubs via endpoint metadata — so there’s nothing to curate.

Also: no account, no cloud sync, no seat count. Everything runs on your machine. Some shops can’t use Postman at all because of the data-residency policy — Bowire has no such constraint.

Scalar is an excellent Swagger UI replacement for REST + OpenAPI. If all you need is pretty REST docs, use Scalar. Bowire steps in when you also have gRPC, SignalR, WebSocket, MQTT, or custom protocols in the same service stack — none of which Scalar covers.

Single-protocol tools. Great at their one thing. Bowire gives you the same Reflection-driven UX for gRPC, and you don’t need a different tool for the REST endpoint sitting next to it, or for the SignalR hub wired into the same service.

Bonus: Bowire’s streaming view is a Wireshark-style append log with filter and pin — hands-down more useful than scrolling through a chat-style drawer on long duplex sessions.

Curl is the right tool for one-shot invocations from a shell. It falls apart the moment you want to browse a service’s surface, round-trip typed request bodies, or handle duplex streams. Bowire covers the “I need to explore this service” case that shells were never built for — and still exports every call as a curl / HTTPie snippet when you need one.

You can — but you’ll rebuild auth, form generation, response viewers, streaming channels, history, environment switching, and recording. Bowire is one MapBowire() line in Program.cs and you get all of it, wired to whichever protocols you’ve installed. Apache 2.0, so fork it if you ever outgrow it.
