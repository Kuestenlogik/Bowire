Beyond the workbench

Five subcommands that don’t open a browser tab. Security scans, fuzzing, JWT tampering, AI-agent integration, intercepting proxy — the Unix-style CLI surface that lives next to bowire --url.

Multi-protocol security scanner. Vulnerability templates + built-in passive checks (TLS, headers, version disclosure) for any HTTP-class target. Emits SARIF 2.1.0 so findings drop straight into GitHub Code Scanning, GitLab Security Dashboard, or Azure DevOps.

bash 
# Scan a target against the community vuln-db corpus + built-in checks
bowire scan --target https://api.example.com \
            --corpus ~/.bowire/vulndb \
            --out findings.sarif \
            --severity medium

Schema-aware field-level fuzzer. Knows not to throw SQL-injection payloads at a latitude field; image.bytes fields get magic-byte mutation, not XSS strings. Right-click in the workbench, or one-shot from the CLI.

bash 
# Categories: sqli / xss / pathtrav / cmdinj — built-in wordlists,
# baseline-diff oracle, schema-aware skip for type-mismatched fields
bowire fuzz --target https://api.example.com \
            --template recordings/order-create.bwf \
            --field '$.customer.email' \
            --payloads xss

Decode, tamper, re-sign JWT tokens from the terminal. The classic alg:none downgrade + claim-swap + weak-secret toolkit, scriptable enough for CI gates and security-regression suites.

bash 
# Decode the header + payload, flag well-known weaknesses
bowire jwt decode <token>

# alg:none downgrade (CVE-2015-9235 class)
bowire jwt tamper <token> --alg-none

# Inject a claim, re-sign with a weak secret to probe verification
bowire jwt tamper <token> \
                  --set isAdmin=true \
                  --secret weak123

Expose Bowire itself as an MCP server. AI agents (Claude Desktop, Cursor, Copilot) drive discovery / invoke / record / mock through their conversation. Stdio transport for desktop wiring, HTTP for over-the-wire clients.

bash 
# Stdio — what Claude Desktop / Cursor / Copilot want
bowire mcp serve --bind stdio

# HTTP / Streamable-HTTP for over-the-wire MCP clients
bowire mcp serve --bind http --port 5097

Note the difference from the --enable-mcp-adapter flag on the launch stepper’s Container ship: that one wraps discovered target methods as MCP tools. mcp serve exposes Bowire’s own operations.

Intercepting forward proxy with HTTPS MITM (auto-generated CA, on-the-fly leaf-cert minting). Captured flows stream live into the workbench and convert with one click into Bowire recordings — ready to fuzz, mock, scan, or replay.

bash 
# Start the proxy + a sidecar API for the workbench Proxy tab
bowire proxy --port 8888 --api-port 8889

# Export the auto-generated CA so the client trusts the MITM leaf certs
bowire proxy --export-ca ~/Downloads/bowire-ca.crt
