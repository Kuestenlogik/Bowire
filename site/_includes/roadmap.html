A short look at what just landed — and what’s on the way.

Recently shipped

Custom domain + full release pipeline v1.0.12

The site moved to bowire.io and the release pipeline now produces every download artefact for a tag — NuGet, MSI, DEB, RPM, multi-arch container on GHCR + Docker Hub — plus auto-publish to Winget and Chocolatey.

Replay recordings as a mock server v0.9.x

bowire mock turns any captured recording into a live endpoint. REST, gRPC (unary + streaming + bidi), WebSocket, SSE, SignalR, Socket.IO, MQTT, GraphQL subscriptions. Hot-reload, chaos injection, schema-only mode (OpenAPI / gRPC / GraphQL), stateful cursors, capture-on-miss.

New protocol plugins plugins

Five sibling plugins joined the family: Akka.NET (mailbox tap), OData, Socket.IO, DIS (IEEE 1278), and UDP. Each ships from its own NuGet package on an independent cadence.

Visual flow editor workbench

Five node types — Request, Condition (with true/false branches), Loop, Delay, Variable — chain into a visual pipeline with response chaining via ${response.X}, import / export as .blf files, and a recursive runner that walks the tree top-to-bottom with live pulse animation.
Next on course
1

Polyglot plugins via sidecar bridge

Run plugins as sidecar processes — JSON-RPC over stdio, the same transport LSP / MCP / DAP settled on for exactly this polyglot problem. Per-language SDKs for Python, Node.js, Go, Rust so a Zenoh, paho-mqtt, NATS-Go, or Temporal plugin doesn’t need to be ported to .NET to plug in.
2

MQTT plugin

MQTT 3.1.1 / 5.0 via MQTTnet. Topics map to services, publish/subscribe map to unary/streaming. Discovery scans $SYS/# or a configured topic prefix. Embedded mode hooks into MQTTnet’s in-process server; standalone connects to any broker via --url mqtt://broker:1883. Closes the IoT gap — no other GUI tool does MQTT + gRPC + REST in one surface.
3

Kafka topics + Schema Registry

Apache Kafka via Confluent.Kafka. Confluent Schema Registry feeds the topic / Avro / Proto / JSON-Schema tree directly into Bowire’s service / method model. Cluster ↔ server, topic ↔ method, produce ↔ unary, consume ↔ server-streaming with per-partition offset selection. The biggest gap in Bowire’s current event-streaming coverage.
4

Surgewave tap-stream browser

Surgewave topics map to services, partitions / consumers map to methods, the tap envelope (event, topic, partition, offset, principal, reason) is already recognised by Bowire’s frame pane. Sibling repo + plugin scaffolding ready — blocked on the Kuestenlogik.Surgewave.Client SDK going public.
5

More protocols: SOAP, NATS, Connect (Buf)

SOAP 1.1/1.2 with WSDL discovery for the enterprise legacy that still runs your invoicing. NATS core + JetStream for cloud-native messaging. Connect (Buf) as a wire-format flag on the existing gRPC plugin — same schema, same discovery, just a different framing. Plus AMQP and a generic JSON-RPC browser on the same wave.
6

Full MCP spec + AI-driven testing

Expose Bowire’s own discovery / invoke / record primitives as MCP tools so an agent can chain discover → invoke → assert → switch env → replay. Ship the full MCP spec — resources + prompts alongside tools, plus SSE / HTTP transport so remote clients like Cursor and Copilot can connect without stdio.

Nothing on this list is a commitment — Bowire is built in the open, and priorities shift with what users actually hit in practice. If something here matters to you, file an issue or start a discussion.
