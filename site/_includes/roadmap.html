On course

A short look at what just landed — and what’s on the way.

Recently shipped

AsyncAPI discovery source + Nuclei template runner v1.5.0

Point Bowire at an asyncapi.yaml / .json document (2.x + 3.0) and channels + operations show up in the sidebar; invocations route through whichever wire plugin the doc’s bindings: declare (MQTT today, Kafka / WebSocket follow). Plus bowire scan --nuclei <dir> reads the 8000+ projectdiscovery/nuclei-templates corpus into the existing SARIF-emitting scan engine.

AMQP plugin — skeleton release v0.1.0

Kuestenlogik.Bowire.Protocol.Amqp sibling repo is live. Single plugin id covers both AMQP 0.9.1 (RabbitMQ, ActiveMQ Classic) and AMQP 1.0 (Azure Service Bus, ActiveMQ Artemis); URL scheme picks the wire (amqp:// / amqp1://). IBowireProtocol surface pinned for embedded hosts and the upstream AsyncAPI binding resolver; discovery + invocation land in v0.2.x.

gRPC-Web transport + TacticalAPI plugin v1.2.0

The gRPC plugin now speaks gRPC-Web as well as native HTTP/2 — opt in with grpcweb@ or the X-Bowire-Grpc-Transport metadata header for browser-fronted services and HTTP/1.1-only ingresses. New TacticalAPI sibling plugin (preview) wraps Rheinmetall's situational-awareness gRPC schema with bundled descriptors.

Replay recordings as a mock server v0.9.x

bowire mock turns any captured recording into a live endpoint. REST, gRPC (unary + streaming + bidi), WebSocket, SSE, SignalR, Socket.IO, MQTT, GraphQL subscriptions. Hot-reload, chaos injection, schema-only mode (OpenAPI / gRPC / GraphQL), stateful cursors, capture-on-miss.

The plugin family grew up plugins + workbench

The previous release waves shipped a whole shelf of capabilities at once. Wire plugins: MQTT (3.1.1 + 5.0 via MQTTnet, with broker scanning), Kafka (Confluent.Kafka + Schema Registry decode), Akka.NET (mailbox tap), OData ($metadata + system query options), Socket.IO, DIS (IEEE 1278), UDP. Workbench: the visual flow editor (Request / Condition / Loop / Delay / Variable nodes, .bwf import-export, recursive live-pulse runner).
Up next
1

Visual viewers and editors for message content

Coordinates render on a live map, PNG bytes as an image, audio bytes as audio. The same annotation that picks the viewer on the response side pairs with a request-side editor — drag a pin to fill lat and lon before submit. Works across every transport (gRPC, REST, GraphQL, MQTT, WebSocket, …) by inspecting payload shapes, not by baking extractors into individual plugins. Right-click any field to refine an auto-detected interpretation, persist team-wide via bowire.schema-hints.json.
2

Polyglot plugins via sidecar bridge

Run plugins as sidecar processes — JSON-RPC over stdio, the same transport LSP / MCP / DAP settled on for exactly this polyglot problem. Per-language SDKs for Python, Node.js, Go, Rust so a Zenoh, paho-mqtt, NATS-Go, or Temporal plugin doesn’t need to be ported to .NET to plug in.
3

Connect (Buf) — third gRPC wire mode

Extends the gRPC plugin with Connect support alongside native HTTP/2 and gRPC-Web. URL-prefix connect@ picks the wire; same schema, same discovery, same invoke pane. Phased rollout: Phase 1 unary + JSON / proto bodies + Connect headers and error format, Phase 2 server-streaming with the Connect 5-byte envelope, Phase 3 bidi over HTTP/2. Tests against the connectrpc/conformance suite.
4

AMQP plugin v0.2 + AsyncAPI Phase C bindings

Build on the v0.1.0 skeleton: discovery via RabbitMQ Management HTTP API (0.9.1) and a synthetic Broker service (1.0); invocation via basic.publish (0.9.1) and Session.Send (1.0); streaming via consumer queue / ReceiverLink. Once invocation lands, the AsyncAPI loader's Phase C bindings unlock too: bindings.amqp and bindings.amqp1 route through the new wire plugin without further loader changes.
5

Surgewave tap-stream browser

Surgewave topics map to services, partitions / consumers map to methods, the tap envelope (event, topic, partition, offset, principal, reason) is already recognised by Bowire’s frame pane. Sibling repo + plugin scaffolding ready — blocked on the Kuestenlogik.Surgewave.Client SDK going public.
6

More protocols: SOAP, NATS, Pulsar, JSON-RPC

SOAP 1.1 / 1.2 with WSDL discovery for the enterprise legacy that still runs your invoicing. NATS core + JetStream for cloud-native messaging (unlocks AsyncAPI bindings.nats too). Pulsar for the Apache event-streaming alternative. Generic JSON-RPC browser by generalising the existing MCP JSON-RPC client.
7

Full MCP spec — resources + prompts

The MCP-adapter today exposes Bowire's primitives as MCP tools. Phase 2 adds the other two halves of the spec: resources (read-only data — bowire://recordings/<id>, bowire://environments/<name>, bowire://services/<protocol>) and prompts (canned AI workflows — bowire.smoke-test(url), bowire.regression-hunt(urlA, urlB)). SSE / HTTP transports for remote clients (Cursor, Copilot) without stdio.

Nothing on this list is a commitment — Bowire is built in the open, and priorities shift with what users actually hit in practice. If something here matters to you, file an issue or start a discussion.
