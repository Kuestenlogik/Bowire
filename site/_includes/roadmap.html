On course

A short look at what just landed — and what’s on the way.

AsyncAPI discovery source v1.5.0

Point Bowire at an AsyncAPI 2.x or 3.0 document and channels show up in the sidebar, routed through whichever wire plugin the doc’s bindings declare.

Nuclei template runner v1.5.0

The 8000+ Nuclei community templates now feed the existing SARIF-emitting scan engine via --nuclei.

AMQP plugin skeleton v0.1.0

Sibling repo live; one plugin covers AMQP 0.9.1 and 1.0 via URL scheme. Discovery and invocation land in v0.2.x.

Replay recordings as a mock server v0.9.x

bowire mock turns any captured recording into a live endpoint with the same protocol semantics as the original.

Plugin family grew up over several releases

MQTT, Kafka, Akka.NET, OData, Socket.IO, DIS, UDP and the visual flow editor all landed in the run-up to v1.5.
Up next
1

AMQP plugin v0.2 + AsyncAPI Phase C bindings

Discovery and invocation on top of the v0.1 skeleton. Once they land, AsyncAPI bindings.amqp / bindings.amqp1 light up automatically.
2

Connect (Buf) — third gRPC wire mode

Connect alongside HTTP/2 and gRPC-Web in the existing gRPC plugin. Phased rollout: unary first, then server-streaming, then bidi.
3

Full MCP spec — resources + prompts

The MCP adapter exposes Bowire’s primitives as tools today. The next phase adds the other two halves of the spec: resources for read-only browsing, prompts for canned AI workflows.
4

More protocols: SOAP, NATS, Pulsar, JSON-RPC

WSDL discovery for enterprise legacy, cloud-native messaging via NATS, the Apache event-streaming alternative via Pulsar, and a generic JSON-RPC browser.
5

Surgewave tap-stream browser

Sibling repo and plugin scaffolding ready. Blocked on the Kuestenlogik.Surgewave.Client SDK going public.
6

Polyglot plugins via sidecar bridge

Run plugins as sidecar processes over JSON-RPC stdio, so Python, Go, Rust, or Node.js plugins don’t need to be ported to .NET to plug in.
7

Visual viewers and editors for message content

Coordinates on a live map, PNG bytes as an image, audio bytes as audio — the same annotation that picks the viewer pairs with a request-side editor.

Nothing on this list is a commitment — Bowire is built in the open, and priorities shift with what users actually hit in practice. If something here matters to you, file an issue or start a discussion.
