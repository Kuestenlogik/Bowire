Solutions

How teams ship with Bowire.
One workbench, multiple use cases.

Bowire's core primitives — discovery, recording, replay, schema-awareness — compose into different products depending on what you point them at. The first solution shipping the lane end-to-end is API security testing; more are following on the same primitives.

Available now

The first solution that grew out of the workbench primitives.

  • Security

    live

    Burp Suite for the non-HTTP protocols — schema-aware, Apache-2.0-licensed, with recordings as the vulnerability-template format.

    Built into the bowire scan subcommand: predicate engine, SARIF 2.1.0 output for GitHub Code Scanning / GitLab / Azure DevOps, JWT toolkit, intercepting proxy with HTTPS MITM, and a reusable scan-template.yml GitHub Action that consuming repos pull in via uses:.

Coming soon

Each one rides on primitives that are already in core — the work is in the productised UX layer on top.

  • Service mocking & virtualization

    Turn a recording into a deterministic stand-in for the live service it captured — same wire shape, same streaming cadence, no upstream needed. Tests run offline, CI doesn't pay the per-run cost of a real broker / database / third-party API. The recording-replay core already does this for one client at a time; productising it means a long-running mock server, latency / error-rate dials, and a multi-tenant control plane.

  • Recording-driven regression tests

    A recording captured against today's service replays as a contract test against tomorrow's. The same envelope + assertion DSL the visual flow editor already runs in the workbench, lifted into a headless bowire replay CLI that returns a green/red exit code for CI. Cross-protocol fan-out comes free: one suite covers gRPC + REST + MQTT + SignalR.

  • Multi-protocol discovery for QA

    An auditor-friendly read-only mode of the workbench that surfaces every endpoint Bowire's plugins can see — gRPC services, REST routes, GraphQL fields, SignalR hubs, MQTT topics — with last-seen timestamps and per-endpoint sample payloads. The discovery loop is already in core; the QA-facing surface is "frozen view, exportable inventory, no invoke".

  • Frame-semantics for geospatial data

    The frame-semantics framework that landed in v1.3 already plots GeoJSON / WKT / DIS PDUs on a MapLibre canvas as soon as the workbench sees them — no annotation work needed for the common shapes. The solution layer on top: vertical templates for tactical / maritime / aviation / logistics dashboards, with the recording-replay core feeding deterministic playback.
