Every shape Bowire can take

Bowire ships as a workbench for testing APIs, but the *shape* it takes depends on where you put it. Five modes ship today; a handful more are being designed.

Shipping today

Run any of these against any service, in any of Bowire's 12 protocols.

Sailboat

Embedded endpoint

No engine of its own — sails with the host's wind. Bowire embeds into your ASP.NET app's process. app.MapBowire() and every endpoint is browsable at /bowire next to your own routes.

Speedboat

Standalone CLI

Installed as a global dotnet tool on your laptop. Point Bowire at any URL — staging gRPC in the morning, REST after lunch, WebSocket all afternoon.

Dazzle ship

Replay mock

Looks like the real backend, isn't. Replay a recorded session as a standalone mock server. Pin responses, ship to CI, no live host required.

Container ship

Docker sidecar

Bowire runs as its own container next to your services. A docker-compose service or a Kubernetes pod sidecar. Same workbench UI as the standalone CLI, every endpoint browsable from inside the cluster.

Cruise ship

Multi-user host

One Bowire, several authenticated passengers. Per-user state across sessions. Today: works behind your OIDC reverse proxy. Tracking the path to GA →

Submarine

Transparent proxy

Sits beneath the wire — sees every call your client makes to your server, in any of Bowire's 12 protocols, without changing either end. Watch, record, optionally modify in flight. Concept →

concept

Ferry

CI/CD runner

Saved collection or recorded session → headless run with assertions → JUnit / JSON report. bowire ferry run in your pipeline, no workbench, just exit codes. Multi-protocol from day one. Concept →

concept

Tugboat

IDE extension

Bowire in a VS Code side panel or a JetBrains tool window. Workspace-aware collections + envs live next to the code (git-friendly), same workbench you know. Concept →

concept

Lighthouse

Scheduled probes + alerts

Long-running probe loop. Saved invocations + assertions + a schedule → outcome ledger + OTLP metrics + Slack / PagerDuty signals on pass ↔ fail. Multi-protocol Postman Monitors. Concept →
